## CPU vs GPU (discussion requested by the user)

The only GPU-shaped kernel is the per-radius modular GEMM `N = XбөҖВ·W` (u32 residues). On
96 cores it costs: n=40 ~40 ms/radius for 1 union (memory-bandwidth-bound table scan),
~1 s for a 100-union batch; n=45 ~0.2 s / ~5.7 s. Everything else вҖ” MN table generation
(add/sub-only, ~1.5вҖ“3 s at n=45), exact bigint certification, stopping logic вҖ” is CPU
work. So for n вүӨ 45 the CPU-first choice is strictly right; a GPU would idle behind the
CPU stages. GPU pays off only for n вүҘ 50 fused sweeps or very large union batches.
Non-Rust GPU routes (CuPy/PyTorch int64 matmul, TPU/XLA) were considered and rejected:
cuBLAS has no modular/integer GEMM, float64 GEMM is exact only for p вүІ 2ВІвҒ¶ (accumulation
bound 2вҒөВі/pВІ вүҘ 1 fails at p вүҲ 2ВівҒ°), and hostвҶ”device residue shuttling through Python
dominates. The clean seam is the in-process `TransformBackend` trait (below) with a
pre-specified GPU arithmetic regime: primes < 2ВівҒ°, u64 accumulators, mandatory reduction
every вүӨ 16 MACs (16В·(2ВівҒ°вҲ’1)ВІ + carry < 2вҒ¶вҒҙ вҖ” stated strictly, per critique finding 10).
CPU regime: primes < 2ВіВ№, u128 accumulation (R вүӨ p(50) = 204,226 < 2В№вҒё terms Г— 2вҒ¶ВІ < 2вҒёвҒ°
вүӘ 2В№ВІвҒё вҖ” proven, `const_assert!`ed).

## Core design

### Partitions and canonical order
- Order `lex_desc_full_parts_v1`: weakly-decreasing part lists, lexicographically
  descending; index 0 = `[n]`, last = `[1^n]` (identity type). Generated directly by the
  descending-parts recursion; backend-free; versioned by name + blake3 hash of the
  encoded list. Every artifact embeds name + hash; mismatch = hard refusal on load.
- `PartitionIndex` also provides rank/unrank at every level m вүӨ n via a `Cnt[m][l]`
  counting table (no hash maps in hot paths); `z_О»`, class sizes `n!/z_О»` (bigint +
  residues), signs, transpose involution as an index permutation. Distances as `u16`.
- Beta-set bitmask representation (2 u64 words for n вүӨ 50) for all rim-hook work.
- `CycleTypeTemplate`: parts-without-fixed-points, `pad(n)` вҶ’ full partition; typed
  errors for does-not-fit (job skip, not fatal); identity class rejected as generator
  unless `allow_identity_generator` (spec В§5.3); split A_n classes are structurally
  unrepresentable (inputs are S_n cycle types) вҖ” documented per spec В§17.2.

### MurnaghanвҖ“Nakayama evaluator (`chars::mn`)
- **Level operators as precompiled CSR**: `M_l : partitions(m) вҶ’ partitions(m+l)`,
  entries = rim-hook removals via beta-set bit arithmetic, sign `(вҲ’1)^{htвҲ’1}` folded in.
  Exact nnz formula `nnz(M_l: mвҶ’m+l) = lВ·(p(m) + p(mвҲ’l) + вҖҰ)` gives the planner the
  exact generation work up front. Total CSR вүҲ 64 MB at n=45 (verify ~0.5 GB at n=50;
  build per-level on the fly if needed).
- **Column DP over a suffix-sharing trie**: a full character column for target
  ОҪ = (lвӮҒвүҘlвӮӮвүҘвҖҰ) is `M_{lвӮҒ}вҲҳвҖҰвҲҳM_{l_k}(e_вҲ…)` applied smallest-part-first; columns sharing
  cycle-multiset suffixes share DP prefixes. Total table generation вүҲ (2вҖ“4)В·p(n)ВІ
  add/sub updates вҖ” near output-optimal: <0.1 s (n=30), ~0.3вҖ“1 s (n=40), ~1.5вҖ“3 s
  (n=45), ~8вҖ“15 s (n=50) on 96 cores. Rayon: par over trie subtrees (top 2вҖ“3 levels) +
  par_chunks inside the dominant final-edge SpMVs. MN needs NO multiplications mod p.
- **Three modes sharing the walker**: (a) full modular table, multi-prime lanes in one
  traversal; (b) exact (`ExactInt`) single/batched columns on demand вҖ” for certification
  (values вүӨ вҲҡ(n!) вүҲ 2 limbs); (c) exact base-class columns for eigenvalues. Degrees f_ПҒ
  via hook-length formula (exact), permanently cross-checked against mode (b)'s identity
  column вҖ” two independent computations.
- A deliberately naive memoized MN reference lives in `testing::` forever; the trie DP
  is differentially tested against it.

### Arithmetic (`arith`)
- CPU regime `p31-u128-accumulate-v1`: deterministic descending table of primes < 2ВіВ№
  (all > n вүҘ 255 trivially), Barrett `ModCtx` (mul, pow, inv, reduce_u128, reduce_big).
  Default k = 3 resident primes (n вүӨ 45); k = 2 at n = 50 (RAM). GPU regime documented
  but unused on CPU (see CPU vs GPU).
- `ExactInt = num_bigint::BigInt` behind an alias in `arith::exact` вҖ” the only module
  naming a bigint crate. Exact values appear only off the hot path: degrees, class
  sizes, Пү/Оё construction with the spec-В§4 divisibility assert, coefficient bounds,
  certification, validation. Оё^r sizes: ~rВ·logвӮӮ|U| bits (вүӨ ~16 kbit worst case at n=50,
  r=75) вҖ” schoolbook is fine.
- Screening never divides by n!: `a_r вүЎ 0 вҮ” N_r вүЎ 0 (mod p)` since gcd(n!, p)=1. The
  spec-В§9.4 divisibility check is validation-layer only; the hot path compensates with
  the always-on mod-p word-count tripwire + Пү-divisibility asserts (critique finding 17).

### Spectra, active rows, symmetries
- `О©` from t exact MN columns (one per base class); assert `f_ПҒ | |C_О»|В·ПҮ^ПҒ(О»)`
  entrywise. Active rows R = rows with some Пү вү  0, computed on EXACT values only (a
  residue-based test could drop a live row). R auto-closed under transposition. Do not
  assume restriction helps (dense classes have R вүҲ q); restrict only when it does.
- **Transpose pairing (halves table storage and GEMM)**: store representative rows
  (ПҒ вүҘ ПҒвҖІ) + self-transpose rows, RВҪ = (q+s)/2. Per union, per prime, TWO power vectors:
  `P[ПҒ] *= (Оё_even+Оё_odd)`, `PвҖІ[ПҒ] *= (Оё_evenвҲ’Оё_odd)` (mixed-parity-safe, spec В§11.1).
  Weights per radius: paired rows `WвӮҠ = fВ·(P+PвҖІ)`, `WвӮӢ = fВ·(PвҲ’PвҖІ)`; self-transpose rows
  `WвӮҠ = WвӮӢ = fВ·P` (NOT fВ·(P+PвҖІ) вҖ” the double-count trap; also assert PвҖІ = P there since
  self-conjugate characters vanish on odd classes). `N_r(ОҪ) = ОЈ_ПҒ X[ПҒ,ОҪ]В·W_{sgn(ОҪ)}[ПҒ]`.
  Table columns grouped even-sign targets first, then odd (each GEMM call binds one of
  WвӮҠ/WвӮӢ); critique confirmed this algebra exactly.
- Parity filtering: single-parity unions compute only the feasible target parity per
  radius (~2Г— GEMM + certification savings). Radius 0: identity preset at distance 0,
  layer 0 synthetic, spectral loop starts r = 1 (restricted rows invalid at r = 0,
  spec В§7); the full-row r=0 identity test lives only in `validate.rs`.

### `CharacterSource` + `TransformBackend` (the GPU seam)
- `CharacterSource` (push-based sweeps): `ResidentTable` (generate once, hand out
  slices; optional mmap cache), `StreamedMnSource` (regenerate per sweep). Blocks =
  `TableBlock { targets: &[PartitionId], planes per prime, col-major }`, delivered
  concurrently, exactly once per sweep, parity-aligned.
- `TransformBackend { init, begin_radius(r, WВұ), apply_block(&self, block, out),
  end_radius }` вҖ” Doc 1's shape, extended with the parity-split weight pair (critique
  finding 1e resolution). Normative contract (reproduced as rustdoc + docs/gpu_backend.md):
  borrowed inputs, device-side caching keyed by (prime, block_id) allowed and immutable
  within a session; blocks unordered/concurrent, per-block disjoint outputs; bit-exact
  fully-reduced residues (no FP tolerance) вҖ” CPU and future GPU must agree bitwise;
  reduction strategy internal but must carry a proven overflow bound; errors вҶ’ job
  suspension. Ships `CpuReference` (naive oracle) and `CpuBlocked` (tiled, rayon over
  blocks). **Eigen-grouped mode is a separate per-union execution path**
  (`GroupedTransform`), NOT forced through the shared-row batched API (finding 6).

### Engine: radius loop, certification, stopping
Per-union state: `first_hit_even/odd: Vec<i32>` (the exact-support oracle; support_r =
{ОҪ : 0 вүӨ fh[r mod 2][ОҪ] вүӨ r} by the r+2 monotonicity lemma вҖ” append uВ·uвҒ»В№), `visited`
bitset, layer log, modular powers (NOT checkpointed вҖ” recomputed by modexp).

Radius loop (lockstep batch over active unions, columns compacted on completion):
1. `P/PвҖІ *= Оё_evenВұОё_odd` per prime; assemble WВұ ; `begin_radius`.
2. One source sweep; per block, `apply_block`; classify each (target, union):
   any nonzero residue вҶ’ POSITIVE (proof); parity-infeasible вҶ’ ZERO (free);
   same-parity first_hit already set вҶ’ POSITIVE by the r+2 lemma (count these in a
   metric вҖ” NEVER assert residues are nonzero there; a positive can vanish mod all
   primes, critique finding 2); else вҶ’ CANDIDATE.
   Always-on per-prime word-count tripwire: `ОЈ_ОҪ |C_ОҪ|В·N_r(ОҪ) вүЎ n!В·|U|^r (mod p)`.
3. **Certification gate вҖ” every radius, before layer commit** (this is what makes
   reported supports exact and the stopping rule sound). Tiers per candidate (ОҪ, r, j):
   T1 bound-zero: `a_r вүӨ вҢҠ|U|^r/|C_ОҪ|вҢӢ = 0` вҮ’ zero (free; from the word-count identity);
   T2 resident CRT: if вҲҸ resident primes > that bound, all-zero residues determine 0
   (3Г—31-bit primes cover bounds < 2вҒ№Ві);
   T3 exact evaluation вҖ” two rigorous variants, chosen empirically (finding 12), both
   implemented: (a) dynamic extra primes with fresh MN column mod pвҖІ until product
   exceeds the bound or a nonzero residue appears; (b) exact bigint dot
   `ОЈ f_ПҒ ПҮ^ПҒ(ОҪ) Оё_ПҒ^r` using a batched exact column (mode b) cached per ОҪ and SHARED
   across unions/radii, incremental exact Оё-powers. Every tier terminates with a
   rigorous verdict; no probabilistic step. Certified-positive surprises
   (`hidden_positives_found`) trigger the repair path: pull first_hit back, re-certify
   below, loop вҖ” distances of other types unaffected.
4. Layer commit: exact support_r вҶ’ update first_hit/new/visited; push LayerRecord.
   Stopping (finding 4 resolution): PRIMARY = spec В§5.2 empty-`new` rule on exact
   supports; EARLY EXIT = `visited вҠҮ parity-feasible upper set` (justified purely as a
   parity bound вҖ” sound at all n incl. SвӮ„/VвӮ„); the normal-closure prediction (A_n/S_n
   for n вүҘ 5) runs as an always-on cross-check assert, never as the termination rule.
   Diagnostic-only abort at r > 4n.
5. Checkpoint; deadline check.

### n вүҘ 50 strategy (stretch)
Mode selection is data-driven at plan time from exact |E_U| = #distinct Оё values
(computed from exact О©; grouping residues is forbidden вҖ” collisions):

  the content-sum range) вҶ’ **H_Оұ eigen-grouping**: one streamed generation sweep

- n=50 risk item: certification tail for huge classes (~1.5вҖ“2 h) вҮ’ intra-layer

## Testing & validation (rigorous per requirement)

Trust chain: brute-force BFS over raw permutations (n вүӨ 10) вҶ’ exact bigint spectral
engine (n вүӨ ~16) вҶ’ modular engine with certification (all n); each layer validated
against the one below plus closed-form anchors valid at ANY n. Tiers: T0 (<2 min, every
commit), T1 (<15 min, release + debug-assertions, every commit), T2 (nightly/manual,
`#[ignore]`, cargo-nextest sharded).

Key families (full tables of named tests in the testing design doc):
- **Unit** (T0): partition count vs OEIS A000041 (n вүӨ 50), canonical-order properties,
  rank/unrank, template padding, transpose involution, self-transpose count = distinct
  odd parts, z_О» hand values, ОЈ class sizes = n!, sign vs materialized permutation,
  degrees vs hardcoded tables, ОЈ fВІ = n!, f_ПҒ = f_ПҒвҖІ, hook degrees вүЎ MN identity column.
- **Character** (T0/T1): trivial/sign rows, transposition column = content-sum closed
  form, n-cycle column nonzero exactly on hooks (Вұ1), transpose-sign relation,
  self-transpose vanishes on odd classes, row/column orthogonality (exact bigint),
  modular вүЎ exact reduced, trie DP вүЎ naive memoized reference.
- **Transform invariants** (spec В§9, T0/T1) over a deterministic union catalog (all
  single classes n вүӨ 7, all 2-class unions n вүӨ 6, seeded random n вүӨ 12, specials:
  `[2]`, `[3]`, `[2]вҲӘ[3]` mixed, `[2,2]` in SвӮ„ вҶ’ VвӮ„ disconnected, n-cycle, empty):
  r=0 identity with FULL rows + negative test that restricted rows fail r=0; r=1
  indicators (class/union/weighted); word-count sum = |U|^r; divisibility; nonnegativity;
  parity filter + its mixed-union negative test; no-new-types-after-stop (+2 radii);
  union-square expansion (K_A+K_B)ВІ; eigen-grouped вүЎ ungrouped; batched вүЎ one-by-one
  (incl. after compaction); **paired вүЎ unpaired transform incl. the self-transpose
  WвӮҠ=WвӮӢ=fВ·P trap** (finding 16a).
- **Brute force** (T1 n вүӨ 8, T2 n = 9вҖ“10): BFS distances asserted constant on classes,
