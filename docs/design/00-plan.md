## User decisions (fixed requirements)

1. **Scale, phased**: n ≤ 30 simple and reliable first; n = 35–45 the real target;
   n ≥ 50 stretch goal the architecture must not preclude.
   (p(30)=5,604; p(35)=14,883; p(40)=37,338; p(45)=89,134; p(50)=204,226.)
2. **Output = support/distances only** — no factorization counts. Hot path = modular
   screening (primes > n); exact big-integer work only to certify candidate zeros.
   Stopping requires exact certified support at every radius (spec §5.2/§12, F4/F9).
3. **Per-layer records**: newly-reached cycle types AND exact-length supports, distances,
   diameter, parity/reachability metadata. Versioned JSON embedding the explicit
   partition order + full metadata (spec §19.3).
4. **Input**: range of n (or auto-selected under time/memory budget) + classes/unions as
   cycle types WITHOUT fixed points, padded per n (`[3]` = 3-cycles for every n ≥ 3).
5. **Characters**: pure-Rust Murnaghan–Nakayama evaluator, validated against
   SymPy-generated fixtures committed to the repo. No GAP/Sage at runtime.
6. **Hardware**: CPU-first (rayon). Primary tier pinned: Kaggle TPU VM used as a CPU box
   (~96 vCPU, ~330 GB RAM, ~70 GB disk, hard ~2 h wall limit) ⇒ checkpoint/resume is
   first-class. Runtime memory gating stays authoritative on smaller machines.
   Transform behind a documented backend trait for a later GPU (cudarc/wgpu) drop-in.
7. **Dev machine Windows 11** ⇒ pure-Rust bigints; no GMP dependency.
8. **Testing**: rigorous and extensive (spec §9 invariants, brute-force BFS oracle,
   SymPy fixtures, property tests, adversarial modular tests).

## CPU vs GPU (discussion requested by the user)

The only GPU-shaped kernel is the per-radius modular GEMM `N = Xᵀ·W` (u32 residues). On
96 cores it costs: n=40 ~40 ms/radius for 1 union (memory-bandwidth-bound table scan),
~1 s for a 100-union batch; n=45 ~0.2 s / ~5.7 s. Everything else — MN table generation
(add/sub-only, ~1.5–3 s at n=45), exact bigint certification, stopping logic — is CPU
work. So for n ≤ 45 the CPU-first choice is strictly right; a GPU would idle behind the
CPU stages. GPU pays off only for n ≥ 50 fused sweeps or very large union batches.
Non-Rust GPU routes (CuPy/PyTorch int64 matmul, TPU/XLA) were considered and rejected:
cuBLAS has no modular/integer GEMM, float64 GEMM is exact only for p ≲ 2²⁶ (accumulation
bound 2⁵³/p² ≥ 1 fails at p ≈ 2³⁰), and host↔device residue shuttling through Python
dominates. The clean seam is the in-process `TransformBackend` trait (below) with a
pre-specified GPU arithmetic regime: primes < 2³⁰, u64 accumulators, mandatory reduction
every ≤ 16 MACs (16·(2³⁰−1)² + carry < 2⁶⁴ — stated strictly, per critique finding 10).
CPU regime: primes < 2³¹, u128 accumulation (R ≤ p(50) = 204,226 < 2¹⁸ terms × 2⁶² < 2⁸⁰
≪ 2¹²⁸ — proven, `const_assert!`ed).

## Core design

### Partitions and canonical order
- Order `lex_desc_full_parts_v1`: weakly-decreasing part lists, lexicographically
  descending; index 0 = `[n]`, last = `[1^n]` (identity type). Generated directly by the
  descending-parts recursion; backend-free; versioned by name + blake3 hash of the
  encoded list. Every artifact embeds name + hash; mismatch = hard refusal on load.
- `PartitionIndex` also provides rank/unrank at every level m ≤ n via a `Cnt[m][l]`
  counting table (no hash maps in hot paths); `z_λ`, class sizes `n!/z_λ` (bigint +
  residues), signs, transpose involution as an index permutation. Distances as `u16`.
- Beta-set bitmask representation (2 u64 words for n ≤ 50) for all rim-hook work.
- `CycleTypeTemplate`: parts-without-fixed-points, `pad(n)` → full partition; typed
  errors for does-not-fit (job skip, not fatal); identity class rejected as generator
  unless `allow_identity_generator` (spec §5.3); split A_n classes are structurally
  unrepresentable (inputs are S_n cycle types) — documented per spec §17.2.

### Murnaghan–Nakayama evaluator (`chars::mn`)
- **Level operators as precompiled CSR**: `M_l : partitions(m) → partitions(m+l)`,
  entries = rim-hook removals via beta-set bit arithmetic, sign `(−1)^{ht−1}` folded in.
  Exact nnz formula `nnz(M_l: m→m+l) = l·(p(m) + p(m−l) + …)` gives the planner the
  exact generation work up front. Total CSR ≈ 64 MB at n=45 (verify ~0.5 GB at n=50;
  build per-level on the fly if needed).
- **Column DP over a suffix-sharing trie**: a full character column for target
  ν = (l₁≥l₂≥…) is `M_{l₁}∘…∘M_{l_k}(e_∅)` applied smallest-part-first; columns sharing
  cycle-multiset suffixes share DP prefixes. Total table generation ≈ (2–4)·p(n)²
  add/sub updates — near output-optimal: <0.1 s (n=30), ~0.3–1 s (n=40), ~1.5–3 s
  (n=45), ~8–15 s (n=50) on 96 cores. Rayon: par over trie subtrees (top 2–3 levels) +
  par_chunks inside the dominant final-edge SpMVs. MN needs NO multiplications mod p.
- **Three modes sharing the walker**: (a) full modular table, multi-prime lanes in one
  traversal; (b) exact (`ExactInt`) single/batched columns on demand — for certification
  (values ≤ √(n!) ≈ 2 limbs); (c) exact base-class columns for eigenvalues. Degrees f_ρ
  via hook-length formula (exact), permanently cross-checked against mode (b)'s identity
  column — two independent computations.
- A deliberately naive memoized MN reference lives in `testing::` forever; the trie DP
  is differentially tested against it.

### Arithmetic (`arith`)
- CPU regime `p31-u128-accumulate-v1`: deterministic descending table of primes < 2³¹
  (all > n ≥ 255 trivially), Barrett `ModCtx` (mul, pow, inv, reduce_u128, reduce_big).
  Default k = 3 resident primes (n ≤ 45); k = 2 at n = 50 (RAM). GPU regime documented
  but unused on CPU (see CPU vs GPU).
- `ExactInt = num_bigint::BigInt` behind an alias in `arith::exact` — the only module
  naming a bigint crate. Exact values appear only off the hot path: degrees, class
  sizes, ω/θ construction with the spec-§4 divisibility assert, coefficient bounds,
  certification, validation. θ^r sizes: ~r·log₂|U| bits (≤ ~16 kbit worst case at n=50,
  r=75) — schoolbook is fine.
- Screening never divides by n!: `a_r ≡ 0 ⇔ N_r ≡ 0 (mod p)` since gcd(n!, p)=1. The
  spec-§9.4 divisibility check is validation-layer only; the hot path compensates with
  the always-on mod-p word-count tripwire + ω-divisibility asserts (critique finding 17).

### Spectra, active rows, symmetries
- `Ω` from t exact MN columns (one per base class); assert `f_ρ | |C_λ|·χ^ρ(λ)`
  entrywise. Active rows R = rows with some ω ≠ 0, computed on EXACT values only (a
  residue-based test could drop a live row). R auto-closed under transposition. Do not
  assume restriction helps (dense classes have R ≈ q); restrict only when it does.
- **Transpose pairing (halves table storage and GEMM)**: store representative rows
  (ρ ≥ ρ′) + self-transpose rows, R½ = (q+s)/2. Per union, per prime, TWO power vectors:
  `P[ρ] *= (θ_even+θ_odd)`, `P′[ρ] *= (θ_even−θ_odd)` (mixed-parity-safe, spec §11.1).
  Weights per radius: paired rows `W₊ = f·(P+P′)`, `W₋ = f·(P−P′)`; self-transpose rows
  `W₊ = W₋ = f·P` (NOT f·(P+P′) — the double-count trap; also assert P′ = P there since
  self-conjugate characters vanish on odd classes). `N_r(ν) = Σ_ρ X[ρ,ν]·W_{sgn(ν)}[ρ]`.
  Table columns grouped even-sign targets first, then odd (each GEMM call binds one of
  W₊/W₋); critique confirmed this algebra exactly.
- Parity filtering: single-parity unions compute only the feasible target parity per
  radius (~2× GEMM + certification savings). Radius 0: identity preset at distance 0,
  layer 0 synthetic, spectral loop starts r = 1 (restricted rows invalid at r = 0,
  spec §7); the full-row r=0 identity test lives only in `validate.rs`.

### `CharacterSource` + `TransformBackend` (the GPU seam)
- `CharacterSource` (push-based sweeps): `ResidentTable` (generate once, hand out
  slices; optional mmap cache), `StreamedMnSource` (regenerate per sweep). Blocks =
  `TableBlock { targets: &[PartitionId], planes per prime, col-major }`, delivered
  concurrently, exactly once per sweep, parity-aligned.
- `TransformBackend { init, begin_radius(r, W±), apply_block(&self, block, out),
  end_radius }` — Doc 1's shape, extended with the parity-split weight pair (critique
  finding 1e resolution). Normative contract (reproduced as rustdoc + docs/gpu_backend.md):
  borrowed inputs, device-side caching keyed by (prime, block_id) allowed and immutable
  within a session; blocks unordered/concurrent, per-block disjoint outputs; bit-exact
  fully-reduced residues (no FP tolerance) — CPU and future GPU must agree bitwise;
  reduction strategy internal but must carry a proven overflow bound; errors → job
  suspension. Ships `CpuReference` (naive oracle) and `CpuBlocked` (tiled, rayon over
  blocks). **Eigen-grouped mode is a separate per-union execution path**
  (`GroupedTransform`), NOT forced through the shared-row batched API (finding 6).

### Engine: radius loop, certification, stopping
Per-union state: `first_hit_even/odd: Vec<i32>` (the exact-support oracle; support_r =
{ν : 0 ≤ fh[r mod 2][ν] ≤ r} by the r+2 monotonicity lemma — append u·u⁻¹), `visited`
bitset, layer log, modular powers (NOT checkpointed — recomputed by modexp).

Radius loop (lockstep batch over active unions, columns compacted on completion):
1. `P/P′ *= θ_even±θ_odd` per prime; assemble W± ; `begin_radius`.
2. One source sweep; per block, `apply_block`; classify each (target, union):
   any nonzero residue → POSITIVE (proof); parity-infeasible → ZERO (free);
   same-parity first_hit already set → POSITIVE by the r+2 lemma (count these in a
   metric — NEVER assert residues are nonzero there; a positive can vanish mod all
   primes, critique finding 2); else → CANDIDATE.
   Always-on per-prime word-count tripwire: `Σ_ν |C_ν|·N_r(ν) ≡ n!·|U|^r (mod p)`.
3. **Certification gate — every radius, before layer commit** (this is what makes
   reported supports exact and the stopping rule sound). Tiers per candidate (ν, r, j):
   T1 bound-zero: `a_r ≤ ⌊|U|^r/|C_ν|⌋ = 0` ⇒ zero (free; from the word-count identity);
   T2 resident CRT: if ∏ resident primes > that bound, all-zero residues determine 0
   (3×31-bit primes cover bounds < 2⁹³);
   T3 exact evaluation — two rigorous variants, chosen empirically (finding 12), both
   implemented: (a) dynamic extra primes with fresh MN column mod p′ until product
   exceeds the bound or a nonzero residue appears; (b) exact bigint dot
   `Σ f_ρ χ^ρ(ν) θ_ρ^r` using a batched exact column (mode b) cached per ν and SHARED
   across unions/radii, incremental exact θ-powers. Every tier terminates with a
   rigorous verdict; no probabilistic step. Certified-positive surprises
   (`hidden_positives_found`) trigger the repair path: pull first_hit back, re-certify
   below, loop — distances of other types unaffected.
4. Layer commit: exact support_r → update first_hit/new/visited; push LayerRecord.
   Stopping (finding 4 resolution): PRIMARY = spec §5.2 empty-`new` rule on exact
   supports; EARLY EXIT = `visited ⊇ parity-feasible upper set` (justified purely as a
   parity bound — sound at all n incl. S₄/V₄); the normal-closure prediction (A_n/S_n
   for n ≥ 5) runs as an always-on cross-check assert, never as the termination rule.
   Diagnostic-only abort at r > 4n.
5. Checkpoint; deadline check.

### n ≥ 50 strategy (stretch)
Mode selection is data-driven at plan time from exact |E_U| = #distinct θ values
(computed from exact Ω; grouping residues is forbidden — collisions):
- stored paired table fits RAM (k=2, 167 GB at n=50 on the 330 GB tier) → regime B;
- `|E_U|` small (e.g. transpositions: |E| ≤ 2·C(n,2)+1 = 2451 at n=50, guaranteed by
  the content-sum range) → **H_α eigen-grouping**: one streamed generation sweep
  aggregates `H_α(ν) = Σ_{θ_ρ=α} f_ρ χ^ρ (mod p)` per union (bucket by the PAIR
  (θ_ρ, θ_ρ′) to compose with pairing); afterwards each radius costs |E|·q MACs (~7 ms).
  Corrected accounting (finding 9): wave memory = |E|·q·4·k bytes PER UNION (20-union
  wave at |E|=2451, k=2 → 80 GB); H-build is scatter-bound ≈ 3 min, not GEMM-rate.
- else → **fused all-radii sweep**: all θ^r mod p known up front; one generation sweep
  dots every emitted column against all r_max weight vectors (n=50, 1 union, r_max=75,
  k=2: ~22 s; m=100: ~37 min). Disk streaming REFUTED (regeneration ~10 s beats reading
  83 GB; Kaggle disk can't hold it anyway) — recompute-don't-reread globally.
- n=50 risk item: certification tail for huge classes (~1.5–2 h) ⇒ intra-layer
  checkpoints (finding 5): screened positives + certified-so-far verdicts + remaining
  queue may be checkpointed mid-certification, with the invariant that stopping tests
  and layer commits read only fully-certified state; resume re-enters certification.

## Testing & validation (rigorous per requirement)

Trust chain: brute-force BFS over raw permutations (n ≤ 10) → exact bigint spectral
engine (n ≤ ~16) → modular engine with certification (all n); each layer validated
against the one below plus closed-form anchors valid at ANY n. Tiers: T0 (<2 min, every
commit), T1 (<15 min, release + debug-assertions, every commit), T2 (nightly/manual,
`#[ignore]`, cargo-nextest sharded).

Key families (full tables of named tests in the testing design doc):
- **Unit** (T0): partition count vs OEIS A000041 (n ≤ 50), canonical-order properties,
  rank/unrank, template padding, transpose involution, self-transpose count = distinct
  odd parts, z_λ hand values, Σ class sizes = n!, sign vs materialized permutation,
  degrees vs hardcoded tables, Σ f² = n!, f_ρ = f_ρ′, hook degrees ≡ MN identity column.
- **Character** (T0/T1): trivial/sign rows, transposition column = content-sum closed
  form, n-cycle column nonzero exactly on hooks (±1), transpose-sign relation,
  self-transpose vanishes on odd classes, row/column orthogonality (exact bigint),
  modular ≡ exact reduced, trie DP ≡ naive memoized reference.
- **Transform invariants** (spec §9, T0/T1) over a deterministic union catalog (all
  single classes n ≤ 7, all 2-class unions n ≤ 6, seeded random n ≤ 12, specials:
  `[2]`, `[3]`, `[2]∪[3]` mixed, `[2,2]` in S₄ → V₄ disconnected, n-cycle, empty):
  r=0 identity with FULL rows + negative test that restricted rows fail r=0; r=1
  indicators (class/union/weighted); word-count sum = |U|^r; divisibility; nonnegativity;
  parity filter + its mixed-union negative test; no-new-types-after-stop (+2 radii);
  union-square expansion (K_A+K_B)²; eigen-grouped ≡ ungrouped; batched ≡ one-by-one
  (incl. after compaction); **paired ≡ unpaired transform incl. the self-transpose
  W₊=W₋=f·P trap** (finding 16a).
- **Brute force** (T1 n ≤ 8, T2 n = 9–10): BFS distances asserted constant on classes,
  then per-type distances/layers/supports/diameter vs engine; exact-length supports via
  boolean set-product DP; word counts a_r vs group-algebra DP (u64 + checked_add, n ≤ 7,
  r ≤ 4) — counts are internal-only but everything rests on them; V₄-in-S₄ disconnected
  case; mixed-parity unions through the whole stack.
- **Adversarial modular** (the one catastrophic failure mode — zero residue treated as
  certified zero): injectable prime lists; `tools/find_adversarial.py` commits (n, U, r,
  ν, primes) tuples with a_r > 0 ≡ 0 mod ∏(small primes injected, e.g. {11,13});
  T1 test runs the modular engine on them asserting candidate→certify→positive, correct
  final distances, no stop at the false-empty layer; screened-positive-never-wrong sweep
  (n ≤ 9); synthetic classifier plumbing tests (fallback invocation observable via a
  counting hook); `stopping_gate_requires_certified_layer` unit test; overflow stress:
  worst-case all-(p−1) dots at lengths {p(45), p(50), 10·p(50)} vs BigUint reference,
  block-reduction boundaries K−1/K/K+1/2K+1, `const_assert!` bound formulas.
- **Differential** (T1): modular vs exact engines, n = 5..12 seeded random unions, full
  output equality + every residue ≡ exact mod p.
- **At-scale self-checks** (no fixtures exist at n = 40; debug-assert + `--self-check`):
  mod-p word-count identity per radius; mod-p radius-1 indicator; post-generation table
  checks (identity column ≡ hook degrees mod p, transpose-sign relation, sampled
  orthogonality); analytic anchor: transpositions give d(ν) = n − ℓ(ν), diameter n−1.
- **Property** (proptest; 64 cases T0 / 2048 T2): transpose involution, padding sums,
  class-size sum, sign consistency, modular≡exact MN, batched≡sequential,
  θ linearity, distance monotonicity under union inclusion, determinism across rayon
  thread counts, **resume ≡ uninterrupted** (checkpoint at random radius).
- **Checkpoint/CLI** (T1): kill-mid-run + resume ≡ uninterrupted (run on BOTH OSes —
  Windows process semantics differ); **kill-during-certification + resume re-enters
  certification, cannot stop on a partially-certified layer** (finding 16b); hash/CRC/
  version refusal tests; atomic-write torn-file test; JSON-schema validation of every
  emitted result; golden files for the n ≤ 8 catalog (volatile fields stripped);
  negative test: no word counts anywhere in output.
- **SymPy fixtures**: SymPy likely has no ready S_n character table (verify at
  implementation; if it does, add it as one more certificate layer) — so
  `tools/gen_fixtures.py` implements an independent Python MN evaluator whose output
  must pass a **layered certificate** before being written: identity column = hook
  degrees; transposition column = content closed form; full exact orthogonality
  X·diag(|C|)·Xᵀ = n!·I; transpose-sign relation; trivial/sign rows. This pins the
  table up to symmetries fixed by the certificates — trustworthy without GAP
  (documented in docs/fixtures.md). Full tables n ≤ 14 (15–16 with `--large`), spot
  values + structured/hard columns n = 17..28, degrees/class data n ≤ 30; chunked +
  restartable; pinned SymPy version; explicit partition list embedded (never SymPy's
  native order); order-first assertion in every Rust consumer before any value compare.
- **Benches** (criterion, non-gating): MN table/column gen, modular GEMV/GEMM sweeps,
  exact-fallback pricing, end-to-end small n; plus `classdiam bench-scaling` CLI harness
  emitting `scaling.json` (measured T_gen, per-radius, RSS, fallback counts, |E_U|

## Verification (end-to-end)

1. `cargo test --workspace` (T0+T1): invariants, brute-force cross-checks n ≤ 8,
   fixtures, differentials, checkpoint kill/resume — on Windows dev box; CI adds Linux.
2. `classdiam verify --max-n 9 --certifier both`: built-in oracle suite.
3. `classdiam run -n 6..=12 -u 2 -u 3 -u "3+2,2"` → check against golden files;
   n=6 transpositions must reproduce the critique-verified example (diameter 5,
   d(ν) = 6 − #cycles).
4. `classdiam run -n 30 -u 2 --self-check`: transposition closed form d(ν) = n − ℓ(ν),
   diameter 29, word-count tripwires green, `hidden_positives_found = 0`.
5. Adversarial: run with injected primes {11,13} on committed tuples — engine must
   certify-and-continue, not stop early.
6. `classdiam bench-scaling` at n ≤ 32 → planner constants calibrated; `estimate -n 45`
   sanity vs the memory table. Scale runs at n = 40/45 on the Kaggle TPU VM, including
   a deliberate deadline suspension + `classdiam resume` continuation.
